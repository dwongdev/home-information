{% comment %} Reusable dropdown options for placement target-selects (LocationViews + Collections). Renders LocationViews first as one per Location (ordered by Location.order_id, views within by LocationView.order_id), then a single 'Collections' . Option values are *tagged* -- ``view:`` or ``collection:`` -- so views and collections can share the same select element without colliding on numeric ids. When ``selected_view_id`` / ``selected_collection_id`` are provided, the matching option gets the ``selected`` attribute.
