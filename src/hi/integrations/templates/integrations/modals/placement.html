Progressive disclosure: by default only the top-level decision is visible ("Place all new items in [target]") with an inventory preview underneath ("5 lights · 3 outlets · ..."). The group rows and per-item rows are hidden behind a "Place differently" disclosure. Inside the disclosure, group rows are single-line and each carries its own per-item drill-down disclosure. Three-level inheritance under the hood is unchanged: top default -> group default -> per-item. Empty selections at any level inherit from the level above; explicit "Don't place" at any level prevents placement for that branch. The top level alone offers a "+ New view" option, which creates one fresh LocationView (named after the integration, attached to the operator's current default Location) on Apply. Group/entity overrides to existing views remain in effect even when the top is "+ New view". Existing-view dropdowns are rendered as -per-Location so multi-Location deployments can disambiguate views with shared names ("All", "Default", etc.).
